## D1. Sparse Pauli Dynamics in the Heisenberg picture

We simulate OTOC^(2) = Tr(rho M U^dag B U M U^dag B U) classically
using Sparse Pauli Dynamics (SPD) (Begusic, Gray, Chan, Science
Advances 10, eadk4321, 2024; Begusic & Chan, PRX Quantum 6, 020302,
2025). The observable is evolved backward through the circuit in the
Pauli operator basis:

1. Initialize B as a single-qubit Pauli operator
2. Evolve B(t) = U^dag B U by conjugating through each gate layer
   in reverse order (Heisenberg picture)
3. Compute A = M * B(t) (single-Pauli left-multiply, O(N) terms)
4. Evaluate <0|A^2|0> using Pauli group signature matching

At each layer, Pauli strings exceeding a weight threshold ell are
discarded (truncation). The truncation error is controlled by the
exponential (screening regime) or power-law (post-transition regime)
decay of Pauli coefficients.

## D2. Gate model

Two-qubit gates are iSWAP-like: U = exp(-i theta/2 (XX+YY)) exp(-i phi ZZ),
matching Willow's fSim decomposition (Bermejo et al., arXiv:2604.15427,
Section II.1.3: "entanglement is generated by uniform two-qubit fSim gates
which can be decomposed into an iSWAP gate followed by a CPHASE gate").

Gate conjugation in the Heisenberg picture is implemented via
precomputed 16-entry Pauli-to-Pauli lookup tables (one per unique
(theta, phi) pair), replacing per-term matrix exponentiation with
O(1) table lookups. This yields 20-77x speedup over numerical
conjugation (commit 66f3608).

Single-qubit rotations are applied analytically using the Pauli
multiplication table.

## D3. Circuit construction

Circuits are constructed on 2D grids (rows x cols) with brickwall
structure (Bermejo Section II.1.3: "alternating layers of entangling
two-qubit gates and random single-qubit gates in a brickwall
structure"). Each layer consists of random single-qubit rotations
followed by iSWAP-like gates on a non-overlapping edge subset.

## D4. OTOC^(2) computation

For OTOC^(2) = <0|(M B(t))^2|0>, we compute A = M * B(t) then
evaluate <0|A^2|0> efficiently using Pauli group signature matching:
two Pauli strings P_j, P_k contribute to <0|P_j P_k|0> only when
their product yields all I or Z factors. We group terms by their
{I,Z} vs {X,Y} signature per qubit, reducing the O(N^2) pairing
to within-group matching only.

## D5. Noise model

Depolarizing noise at rate gamma per 2-qubit gate: each Pauli string
with non-identity support on gate qubits is damped by (1-gamma).
Willow-relevant rates: gamma in [0.003, 0.007].

Finding: depolarizing noise does NOT significantly reduce OTOC^(2)
truncation error (12q w<=4: noiseless 27.5% error, gamma=0.005
27.2% error). The attack relies on circuit structure, not noise.

## D6. Validation

Exact state-vector simulation for systems up to 10 qubits provides
ground-truth validation. SPD achieves machine precision (relative
error < 10^{-13}) at max weight = n for all tested systems (4-10
qubits, depths 2-8).

## D7. M-B placement

Following Bermejo et al. Section II.1.3: "we therefore choose M to
be near the edge of the physical lightcone of B, where we observe
a maximum signal size." We test a distance ladder (Manhattan distance
1-4) on 12-qubit 3x4 grids to characterize SPD difficulty vs M-B
separation.

## D8. Convergence and cross-validation

Three independent classical paths provide cross-validation (AGENTS.md
Section D5 requirement):
- Path A (claude4): SPD with fixed weight truncation
- Path B (claude8): Schuster-Yin Pauli-path with weight-bounded truncation
- Path C (claude7): Adaptive top-K Pauli weight truncation

All three paths converge on the screening-regime feasibility verdict
and the post-transition power-law tail characterization.
